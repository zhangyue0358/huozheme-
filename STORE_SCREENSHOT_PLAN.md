# 活着吗应用商店截图计划

更新日期：2026-07-05

目标：先准备一组能用于 App Store / Google Play 首发审核的截图素材。截图使用测试账号数据，不展示真实手机号、真实个人信息或真实好友隐私。

## 截图基调

- 黑绿主视觉，和 App 当前 UI 保持一致。
- 页面内容要像真实使用过，不要空白。
- 文案克制，不做夸大承诺，不暗示医疗、心理咨询、紧急救助能力。
- 截图里不要出现完整手机号，只展示脱敏手机号或测试账号昵称。

## 测试数据建议

可以直接运行脚本准备截图数据：

```bash
npm run prepare:screenshots
```

这个脚本会重置测试账号 A/B 的云端业务数据，生成适合截图的昵称、今日打卡、随笔、照片、三件事、历史日记和好友关系。不要在真实用户账号上运行。

测试账号 A：

- 昵称：张三
- 心情：😊 开心，难得有点亮，就先好好接住。
- 随笔：今天只是普通地过了一天，但也算稳稳落地。
- 箴言：今天不用很厉害，能把自己带到晚上就很好。
- 三件事：
  - 喝水，吃饭，出门走一小段
  - 回复一条重要消息
  - 早点睡

测试账号 B：

- 昵称：李四
- 心情：😐 不好不坏，普通也算认真活了一点。
- 好友状态：今天待确认或已确认都可以按截图需要切换。

## 截图拍摄流程

1. 用 Expo Go 或内测包登录测试账号 A。
2. 设置昵称、心情、天气、随笔、照片、箴言和三件事。
3. 用测试账号 B 添加 A 为好友，接受后测试戳一下和回馈。
4. 回到 A 截首页、随笔、三件事、日历、好友、我的页。
5. 截图前检查：无真实手机号、无真实照片、无红屏/调试提示、无 Expo 开发菜单。

## 本地素材目录

最终截图放到：

- iPhone：`store-assets/screenshots/iphone/`
- Android：`store-assets/screenshots/android/`
- 手机原图备份：`store-assets/screenshots/source/`

推荐命名：

1. `01_home.png`
2. `02_journal_photos_quote.png`
3. `03_todos.png`
4. `04_calendar.png`
5. `05_friends.png`

放好后检查：

```bash
npm run export:screenshots
npm run check:screenshots
```

`export:screenshots` 会把 `source/` 里的原图导出为 App Store Connect 6.9 英寸 iPhone 尺寸：`1290 x 2796`。
同时会导出 Android 版本到 `store-assets/screenshots/android/`，保留原图比例，不裁切不拉伸，只补左右深色边框。

## 上架前仍需确认

- 如果使用 Expo Go 截图，状态栏可能和正式包不同；正式提交前最好用 preview/production 包再截一轮。
- Android 截图要避开系统虚拟按键遮挡。
- iOS 截图要确认安全区和底部导航没有被裁切。
- 商店后台具体尺寸以当时 App Store Connect / Google Play Console 要求为准。
