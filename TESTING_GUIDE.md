# 活着吗真机测试清单

## 0. 启动

```bash
npm start
```

用 Expo Go 扫码打开 App。手机和电脑需要在同一个 Wi-Fi；如果连不上，使用：

```bash
npx expo start --tunnel
```

## 1. 登录

正式内测优先测试手机号验证码登录。登录页底部的 `测试账号 A` / `测试账号 B` 只用于开发排查，绕过短信服务，但仍然走真实 Supabase 数据。

使用测试账号前，先在 Supabase Authentication Users 里手动创建：

- `test-a@huozhema.local`
- `test-b@huozhema.local`

默认开发密码是 `.env` 中的 `EXPO_PUBLIC_DEV_TEST_PASSWORD`，如果没配置则使用 `HuozhemaTest2026!`。
开发测试账号会自动绑定内测手机号：测试账号 A 为 `13900000001`，测试账号 B 为 `13900000002`，用于测试手机号加好友。

如果需要把 A/B 恢复成干净测试状态，不要只卸载 App。卸载只会清本机缓存，不会删除 Supabase 云端好友、日记和照片。可在本机项目根目录运行：

```bash
npm run reset:dev-accounts
npm run check:dev-accounts
```

正式手机号登录流程：

1. 输入手机号。中国手机号可直接输入 11 位，其他地区请带国际区号。
2. 点“发送验证码”。
3. 发送成功后按钮进入 60 秒倒计时，倒计时结束后才可重新发送。
4. 查看短信里的 6 位验证码。
5. 回到 App 输入验证码。
6. 点“确认登录”。
7. 如果重新发送过验证码，只使用最新一条短信里的验证码。

通过标准：进入 App 首页，不再停留在登录页。

如果收不到短信，按顺序检查：

- Supabase Function Logs 是否出现 `SMS OTP sent`。
- 阿里云短信发送记录是否成功。
- Supabase Authentication 的 Phone Provider 和 Send SMS Hook 是否启用并保存。
- Supabase Secrets 是否包含 `AUTH_HOOK_SECRET`、阿里云 AccessKey、签名和模板 Code。
- 阿里云短信签名是否完成运营商实名制报备。

## 2. 单账号功能

- 选择一个心情。
- 点“确认我还活着”。
- 写一段“随笔小记”，点“保存”，看到“保存中/已保存”反馈。
- 点“添加照片”，选择 1 张照片，确认缩略图出现。
- 修改“送给自己的一句话”，点“存”。
- 添加一件今日事项。
- 给事项标记“大事”。
- 切换事项完成状态。
- 点“看看今天的我”，确认内容包含累计存活、心情、随笔、照片数量、箴言、完成事项。
- 到“我”页打开“查看所有日记”，确认至少显示今天 1 篇，并能看到照片缩略图。
- 关闭再重新打开 App，确认今日随笔、照片、三件事和历史日记仍然存在。

## 3. 双账号好友功能

准备两个真实手机号账号 A 和 B，最好两台手机分别登录。开发排查阶段也可使用 `测试账号 A` / `测试账号 B`。

1. A 登录，确认“我”页能看到隐藏手机号 `139****0001`。
2. B 登录，输入 A 的手机号 `13900000001`，发送好友申请。
3. A 重新打开好友页，接受申请。
4. 双方确认好友列表里能看到对方状态。
5. A 点“戳一下”B。
6. B 在“有人戳你”里点“我还活着”。
7. A 等几秒自动同步，确认 B 从“待确认是否活着”进入“今天已确认”，状态显示“还活着”，按钮仍保持“已戳”。

通过标准：戳一下时有反馈文案，不报错；今天已经戳过的好友，重新登录后仍显示“已戳”。

## 4. 异常测试

- 未确认活着前，保存随笔、添加照片、选心情、添加三件事、点“看看今天的我”，都应提示“请先确认还活着吗？”。
- 重复点打卡，不应重复增加今日记录。
- 今日事项超过三件，应不能继续添加。
- 照片达到 3 张后，应不能继续添加。
- 重复添加同一个手机号，应提示已经申请或已经是好友。
- 连续尝试添加不存在的手机号，超过限额后应提示“添加太频繁，请稍后再试”。
- 好友申请发送成功，应显示 App 内提示，不弹系统提示。
- 6 月 11 日当天操作的日记，不应出现在 6 月 10 日。
- 断网后操作，应看到失败提示，不白屏。

## 5. 已知内测边界

- 好友添加目前按手机号提交给后端 RPC 匹配，输入框应显示“输入好友手机号”。昵称只用于好友列表展示。
- 推送通知暂未接入，提醒入口只是设置草稿。
- App 内删除本应用数据会删除业务数据和照片，但保留手机号登录账号，方便用户继续使用。
- App 内注销账户会先提交注销请求并退出登录；后台脚本按保留期处理个人信息、内容数据，并在内容保留期结束后删除 Supabase Auth 用户，最后清理已完成的注销请求。上线前需要配置 GitHub Actions Secrets 并手动运行一次定时任务验证。
